Promises in JavaScript

Promise is a JavaScipt Object that simply links consuming and producing codes.It holds the upcoming value of an async operation. Before ES6 callbacks functions were used to handle async tasks which can lead to a callback hell.Promise helps to avoid that.......

There are three states of Promise.
Unresolved - If the result is not ready then the promise is unresolved.
Resolved - If the result is ready then the promise is resolved
Rejected - If an error occured then promise is rejected

Creating a Promise

const myPromise = new Promise(function(myResolve,myReject){
myResolve(); // When Successful
myReject(); //When Error
});


A new promise is created using Promise constructor. It takes a single argument(a callback), which takes two callbacks, resolve and reject.



Promise Instance Methods

then, catch and finally are three main Promise API methods


Promise then - It notifies when a asynchronous operation is completed, either successful or fail. Multiple promises can be chained together using it.

promise.then(Success);
Promise.resolve()
.then(() => console.log('then#1'))
.then(() => console.log('then#2'))
.then(() => console.log('then#3'));


All promises are resolved and following then are called, so output will be

then#1
then#2
then#3


Promise catch - It takes a function as an argument that handles error if occured. It is never called if everthing goes well

promise.catch(Error);
Promise.resolve()
.then(a)
.catch(console.error)


Promise then - It is only called when the promise is resolved. If a piece of code is to be executed after .then and .catch then it is used

Promise.resolve()
.then(a)
.then(b)
.catch(console.error)
.finally(() => console.log('always called'))

Centre Back
